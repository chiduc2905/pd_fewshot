3.1 Problem Formulation
We consider the standard N-way K-shot episodic few-shot classification setting.
For each episode, the support set is defined as
\mathcal{S}={S_c}_{c=1}^N,S_c={x_c^1,\ldots,x_c^K},

where x_c^kdenotes the k-th support image of class c.
Given a query image q, the goal is to predict its class label y_q\in{1,\ldots,N}.
Let F_\thetadenote the feature extractor. For an input image x, the backbone produces a spatial feature map
F_\theta(x)\in\operatorname{R}^{H\times W\times D}.

We flatten the spatial dimensions and obtain a set of local tokens
T(x)={t_i(x)}_{i=1}^M,t_i(x)\in\operatorname{R}^D,M=H\cdot W.

For convenience, we denote the query token set by
T(q)={u_j}_{j=1}^M,

and the support token set of the k-th shot in class cby
T_c^k={t_{c,k,i}}_{i=1}^M.

Unlike prototype-based methods that summarize each class by a single vector, and unlike flat local-matching methods that merge all support tokens into an unordered descriptor bank, we explicitly preserve the hierarchical structure of support evidence:
	a class is a set of support images, 
	each support image is a set of local tokens. 
Our key idea is to formulate few-shot classification as hierarchical support inference over this nested structure, where the similarity between a query and a class is measured by a support-aware Sliced Wasserstein distance.

3.2 Motivation
Recent metric-based few-shot methods have demonstrated that preserving structured evidence is critical for robust class discrimination. DN4 shows the importance of local descriptors rather than global image embeddings, while DeepEMD formulates few-shot matching as region-level optimal transport. SetFeat further argues that image representation should be treated as a feature set rather than a monolithic vector. However, existing methods still leave an important gap: they either collapse support evidence too early into coarse summaries or perform matching on flattened support structures without explicitly modeling how evidence is distributed across different support shots.
In few-shot classification, a correct class should not only contain locally similar evidence, but should also explain the query in a manner that is consistent across multiple support examples. This observation motivates us to move from flat query-class matching to hierarchical support-aware distributional inference.

3.3 Hierarchical Support Measures
Shot-level empirical measure
For each support image x_c^k, we define a shot-level empirical measure over its local tokens:
\mu_c^k=\sum_{i=1}^{M}a_{c,k,i}\mathrm{\thinsp}\delta_{t_{c,k,i}},

where \delta_tdenotes the Dirac measure centered at token t, and a_{c,k,i}\geq0are normalized token weights satisfying
\sum_{i=1}^{M}a_{c,k,i}=1.

In practice, the weights are generated by a lightweight token scoring module:
{\widetilde{a}}_{c,k,i}=g_\phi(t_{c,k,i}),a_{c,k,i}=\frac{\exp\funcapply({\widetilde{a}}_{c,k,i})}{\sum_{m=1}^{M}{\exp\funcapply(}{\widetilde{a}}_{c,k,m})}.

This allows the model to emphasize informative local evidence while suppressing background or weakly relevant tokens.
Query empirical measure
Similarly, for the query image q, we define
\nu_q=\sum_{j=1}^{M}b_j\mathrm{\thinsp}\delta_{u_j},

where
{\widetilde{b}}_j=g_\psi(u_j),b_j=\frac{\exp\funcapply({\widetilde{b}}_j)}{\sum_{m=1}^{M}{\exp\funcapply(}{\widetilde{b}}_m)}.

Class-level hierarchical representation
Instead of flattening the support class into a single pooled token measure, we preserve the class as a set of shot-level measures:
\operatorname{M}_c={\mu_c^1,\ldots,\mu_c^K}.

This representation is the central object of our method. It explicitly distinguishes:
	within-shot token structure, and 
	across-shot support structure.

3.4 Sliced Wasserstein Class Distance
Preliminaries on Sliced Wasserstein distance
Given two empirical measures \muand \nuin \operatorname{R}^D, the Sliced Wasserstein (SW) distance compares them by projecting both measures onto one-dimensional directions and averaging the corresponding 1D Wasserstein distances:
SW^2(\mu,\nu)=\int_{\mathbb{S}^{D-1}} W_2^2(P_\omega\mu,P_\omega\nu)\mathrm{\thinsp} d\omega,

where P_\omegadenotes the projection onto direction \omega\in\mathbb{S}^{D-1}. In practice, this integral is approximated using Lsampled directions:
SW^2(\mu,\nu)\approx\frac{1}{L}\sum_{\ell=1}^{L}W_2^2(P_{\omega_\ell}\mu,P_{\omega_\ell}\nu).

Compared with full optimal transport, SW offers a favorable tradeoff between distributional geometry and computational efficiency, which is particularly important when token-level comparisons must be performed repeatedly across classes and episodes.
Flat class measure is insufficient
A naive query-class score would compare the query measure \nu_qwith a flattened class measure
μcflat=k=1Ki=1Mαc,k ac,k,i δtc,k,i,

and compute
s_{\mathrm{flat}}(q,c)=-SW(\nu_q,μcflat).

However, such a formulation loses support-shot identity and cannot distinguish whether a low query-class distance arises from:
	broad agreement across multiple support examples, or 
	a single accidental support shot that happens to match the query. 
To address this issue, we introduce a support-aware class inference rule.
3.5 Hierarchical Support Sliced-Wasserstein Inference
Shot-level compatibility
We first compute the query-to-shot distances:
d_{c,k}(q)=SW(\nu_q,\mu_c^k).

These distances indicate how well each support shot individually explains the query.
Support-aware mixture over shots
We then define a query-conditioned shot responsibility distribution:
\gamma_{c,k}(q)=\frac{\exp\funcapply(-d_{c,k}(q)/\tau)}{\sum_{\ell=1}^{K}{\exp\funcapply(-}d_{c,\ell}(q)/\tau)},

where \tau>0is a temperature parameter.
Using these responsibilities, we construct a query-conditioned class measure
μc(q)=k=1Kγc,k(q) μck.

Unlike a fixed class representation, μc(q)is dynamically adapted to the query, but still remains grounded in the hierarchical support decomposition.
Hierarchical class distance
Our hierarchical support-aware Sliced Wasserstein distance is defined as
\mathcal{D}_{\mathrm{HS-SW}}(q,c)=SW(\nu_q,μc(q))+λcons Ωcons(q,c)+λred Ωred(c),

where:
	SW(\nu_q,μc(q))measures query-to-class distributional proximity, 
	\Omega_{\mathrm{cons}}(q,c)is a support-consistency regularizer, 
	\Omega_{\mathrm{red}}(c)is a redundancy penalty on the support set. 
The class logit is then
s(q,c)=-\mathcal{D}_{\mathrm{HS-SW}}(q,c).

Finally, the predicted label is
{\hat{y}}_q=\arg\funcapply{\max\funcapply}\below c s(q,c).

3.6 Support-Consistency and Redundancy Regularization
Support-consistency regularization
A desirable support class should explain the query consistently across multiple support shots rather than relying on a single support outlier. To encode this bias, we define
\Omega_{\mathrm{cons}}(q,c)=-\sum_{k=1}^{K}{\log\funcapply(\epsilon+}\gamma_{c,k}(q)),

where \epsilon>0is a small constant for numerical stability.
This regularizer is minimized when the responsibility mass is not overly concentrated on a single shot. Intuitively, it encourages the decision rule to prefer classes whose support evidence is more broadly consistent with the query.
Support redundancy regularization
If multiple support shots are nearly identical, a class may appear overly confident due to redundant evidence rather than genuinely diverse support coverage. To discourage this effect, we define
\Omega_{\mathrm{red}}(c)=\frac{1}{K(K-1)}k≠lexp⁡ ⁣(-SW(μck,μcl)).

When support shots are highly similar, their pairwise SW distances are small and the redundancy term becomes large. This encourages the model to distribute importance toward complementary support evidence.

3.7 Training Objective
For each query qwith ground-truth label y_q, we optimize the episodic cross-entropy loss:
\operatorname{L}_{\mathrm{cls}}=-\log\funcapply\frac{\exp\funcapply(s(q,y_q))}{\sum_{c=1}^{N}{\exp\funcapply(s(q,c))}}.

The overall training objective is
\mathcal{L}=\operatorname{L}_{\mathrm{cls}}+\beta_{\mathrm{cons}}\operatorname{L}_{\mathrm{cons}}+\beta_{\mathrm{red}}\operatorname{L}_{\mathrm{red}},

where \operatorname{L}_{\mathrm{cons}}and \operatorname{L}_{\mathrm{red}}are optional auxiliary regularization losses aggregated over the episode.
In the simplest implementation, these auxiliary terms are already included inside \mathcal{D}_{\mathrm{HS-SW}}, so the model can be trained using only \operatorname{L}_{\mathrm{cls}}.

3.8 Theoretical Properties
We next state several basic properties of the proposed classifier.
Proposition 1. Permutation invariance over support order
Let \pibe any permutation of the support-shot indices \left\{1,\ldots,K\right\}. Then
\mathcal{D}_{\mathrm{HS-SW}}(q,c;{\mu_c^1,\ldots,\mu_c^K})=\mathcal{D}_{\mathrm{HS-SW}}(q,c;{\mu_c^{\pi(1)},\ldots,\mu_c^{\pi(K)}}).

Proof sketch.
The set of shot-level distances \left\{d_{c,k}(q)}_{k=1}^K\right.is merely reordered under permutation. The softmax-defined responsibilities \gamma_{c,k}(q), the mixture μc(q), and both regularizers \Omega_{\mathrm{cons}}and \Omega_{\mathrm{red}}are symmetric with respect to the support-shot indices. Therefore the final distance remains unchanged. \box
Proposition 2. Reduction to flat SW classifier
If \lambda_{\mathrm{cons}}=\lambda_{\mathrm{red}}=0and \gamma_{c,k}(q)is replaced by fixed uniform weights \gamma_{c,k}=1/K, then
\mathcal{D}_{\mathrm{HS-SW}}(q,c)=SW(\nu_q,μcflat),

which reduces the proposed classifier to a flat query-class SW matching rule.
Interpretation.
This proposition shows that our method strictly generalizes a plain SW classifier.
Proposition 3. Prototype-like degenerate case
Assume each shot-level measure collapses to a single atom,
\mu_c^k=\delta_{z_c^k},\nu_q=\delta_{z_q},

and let the support-consistency and redundancy terms vanish. Then the hierarchical distance reduces to a point-based metric between z_qand a weighted class representative.
Interpretation.
Prototype-based classification appears as a degenerate low-complexity case of the proposed framework.
Proposition 4. Support-consensus bias
For fixed base distance SW(\nu_q,μc(q)), the term \Omega_{\mathrm{cons}}(q,c)is minimized when the shot responsibilities \gamma_{c,k}(q)are more evenly distributed.
Interpretation.
The classifier is explicitly biased toward classes whose evidence is supported by multiple support examples, rather than by a single accidental match.

3.9 Discussion
Our formulation differs from prior structured few-shot methods in three aspects.
First, unlike prototype-based methods, it does not collapse support evidence into a single vector. Second, unlike flat local-matching approaches, it preserves support-shot identity and models support as a hierarchical object. Third, unlike pairwise transport-based methods, it elevates distributional matching to the class level, where the final score depends not only on query-support proximity but also on how consistently the support set as a whole explains the query. This shift from flat matching to hierarchical class inference is the core novelty of the proposed method.

This follows from the symmetry of the shot responsibility distribution, the query-conditioned mixture, and the support regularizers.
Proposition 1. Flat-SW special case
When support consistency and redundancy regularization are removed and all shot weights are uniform, HS-SW reduces to a flat Sliced Wasserstein classifier.
Proposition 2. Prototype special case
When both query and support measures collapse to single atoms, HS-SW reduces to a point-based class distance and thus recovers prototype-like classification.
Proposition 3. Consensus preference
Among classes with identical base query-class SW distance, HS-SW prefers the class whose support responsibility distribution is less concentrated, i.e. whose evidence is better distributed across support shots.

Minimal implementation blueprint
Below is the smallest realistic version.
A. Backbone and tokens
Use your existing backbone, for example Conv-4 or ResNet-12.
# x: [B, C, H, W]
feat = backbone(x)                  # [B, D, Hf, Wf]
tokens = feat.flatten(2).transpose(1, 2)   # [B, M, D]
tokens = F.normalize(tokens, dim=-1)
B. Token weights
A tiny scoring MLP:
class TokenWeighter(nn.Module):
    def __init__(self, dim, hidden=128):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(dim, hidden),
            nn.ReLU(inplace=True),
            nn.Linear(hidden, 1)
        )

    def forward(self, tokens):  # [B, M, D]
        logits = self.net(tokens).squeeze(-1)   # [B, M]
        weights = torch.softmax(logits, dim=-1)
        return weights
Use one shared module for support and query first. Split later only if needed.
C. Shot-level measures
For each support image:
	tokens: MD
	weights: \left[M\right]
Treat that as one empirical measure.
D. Approximate SW
Use Lrandom projections.
def sliced_wasserstein_distance(x, y, wx=None, wy=None, n_proj=32):
    # x: [Mx, D], y: [My, D]
    # wx, wy: [Mx], [My]
    D = x.size(-1)
    proj = torch.randn(n_proj, D, device=x.device)
    proj = F.normalize(proj, dim=-1)  # [L, D]

    x_proj = x @ proj.t()  # [Mx, L]
    y_proj = y @ proj.t()  # [My, L]

    x_proj, _ = torch.sort(x_proj, dim=0)
    y_proj, _ = torch.sort(y_proj, dim=0)

    # simplest equal-mass approximation
    m = min(x_proj.size(0), y_proj.size(0))
    dist = ((x_proj[:m] - y_proj[:m]) ** 2).mean()
    return dist
This is the simplest version. Later you can add weighted 1D transport instead of equal-mass sorting.
E. Query-to-shot distances
For each class c, for each shot k:
d_ck = SW(query_tokens, support_tokens[c][k])
Store all d_{c,k}.
F. Shot responsibilities
gamma = torch.softmax(-d_ck / tau, dim=0)   # over K shots
G. Query-conditioned class score
Two implementation options.
Option 1: mixture in score space
This is easier first:
{\widetilde{d}}_c=\sum_{k}\gamma_{c,k}d_{c,k}

Then add consistency regularizer:
\Omega_{\mathrm{cons}}=-\sum_{k}{\log\funcapply(\epsilon+}\gamma_{c,k})

and redundancy:
precompute support-shot pairwise SW:
\Omega_{\mathrm{red}}(c)=\frac{1}{K(K-1)}\sum_{k\neq\ell}{\exp\funcapply(-SW(}\mu_c^k,\mu_c^\ell)).

Final:
D_c={\widetilde{d}}_c+\lambda_{\mathrm{cons}}\Omega_{\mathrm{cons}}+\lambda_{\mathrm{red}}\Omega_{\mathrm{red}}
s_c=-D_c

This is the most practical starting point.
Option 2: mixture in measure space
Build a query-conditioned class measure using \gamma_{c,k}, then compute one more SW between the query and this mixed class measure. This is closer to the theory, but harder to implement neatly at first.
H. Episode loss
logits = -distances   # [n_query, N_way]
loss = F.cross_entropy(logits, labels)
I. Best minimal ablations
You should at least compare:
	cosine / Proto head 
	flat SW head 
	shot-weighted SW without regularizers 
	shot-weighted SW + consistency 
	shot-weighted SW + consistency + redundancy 
That ablation structure is important because it shows the gain comes from the hierarchical inference law, not just from using SW.

A good starting configuration:
	Backbone: same as your baseline 
	Token normalization: yes 
	Number of projections L: 16 or 32 
	Temperature \tau: 0.2 or 0.5 
	\lambda_{\mathrm{cons}}: 0.05 to 0.2 
	\lambda_{\mathrm{red}}: 0.01 to 0.05 
Start with:
D_c=\sum_{k}\gamma_{c,k}d_{c,k}+\lambda_{\mathrm{cons}}\Omega_{\mathrm{cons}}

and add redundancy only if needed.

How to present the novelty in one paragraph
You can use this almost directly:
We propose Hierarchical Support Sliced-Wasserstein Inference (HS-SW), a few-shot classifier that models each support class as a set of shot-level empirical measures rather than a prototype or a flat descriptor pool. Given a query measure, HS-SW first estimates query-conditioned shot responsibilities based on shot-wise Sliced Wasserstein distances, then performs class inference using a support-aware hierarchical distance regularized by support consistency and redundancy structure. In contrast to prior metric-based methods that either collapse class evidence too early or ignore support hierarchy during matching, HS-SW introduces a class-level inference rule that explicitly favors support sets whose evidence is both distributionally close to the query and consistently expressed across multiple support examples.
